About:
-------------------------------------------------------------------------------
TauPsi is a typst template to create presentations suited for academical use.
It is heavily inspired by functional programming.
The concepts / abstractions used to program presentations are

1. Slides
2. Descriptor / Description
3. Description Arithmetics
4. Containers

Ad 1: Slides
~~~~~~~~~~~~
A single slide is generated by the slide function
  slide :: string -> string -> colour_palette -> content -> content
which takes the section, subsection, colour_palette and content
and presents the content in small frame showing the section and subsection,
the number of the slides as well as authors name and affiliations.
The 'stepping behaviour' of presentations that the slides are build up
gradually is implemented through *descriptions*.

2. Descriptor / Description
~~~~~~~~~~~~~~~~~~~~~~~~~~~
A description is used to generate slides which gradually build up.
It is
  description :: (N, (N -> content))
a tuple of the maximal number of steps and a function which maps
an integer number to content.
The TauPsi framework provides *descriptors* which are functions that
return descriptions.

2.1 list_descriptor :: [(string, [content])] -> description
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Used to describe a list of items.
The first element of each tuple is a modifier and has to be either "S", "B"
or "M", which stands for single, block or modify.
An element with a "S" modifier describes one item to be shown.
An element with a "B" modifier represents a block of multiple items.
An element with a "M" modifier represents a single item which is extended
or modified multiple times.
For instance does
list_descriptor((
  ("S", [A]),
  ("S", [B]),
  ("B", ([C], [D], [E])),
  ("M", ([F], [FG], [FGH]))
))
describe a list which

A -> A -> A -> A -> A  -> A
     B    B    B    B     B
          C    C    C     C
          D    D    D     D
          E    E    E     E
               F    FG    FGH

where each arrow to the side marks a new step.

2.2 image_scatter_descriptor :: [(string, length, length, length, length)] -> description
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Used to describe freely placed images.
The parameters inside the tuple are path, width, height, posx, posy of the image.
I is allowed to set an element of none which does not shown anything.
This can be used to bloat the description to the same size
of other descriptions.

Example:
  image_scatter_descriptor((
    (path1, width1, height1, posx1, posy1),
    (path2, width2, height2, posx2, posy2)
  ))


2.3 equation_descriptor :: [content] -> description
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Used to describe an equation which is gradually build up or modified.
Each element has to contain the full equation as it is to be shown.

Example:
  equation_descriptor((
    [$ 1 $],
    [$ 1 + 2 $],
    [$ 1 + 2 + 3 $]
  ))

2.4 equation_descriptor2 :: [content] -> description
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Used to describe an equation which is gradually build up or modified.
In this version of the equation_descriptor each elemen has to contain
the part which is added to the equation. I.e. to obtain the same
result as in the example of 2.3

  equation_descriptor2((
    [$ 1 $],
    [$ + 2 $],
    [$ + 3 $]
  ))

has to be used.

3. Description Arithmetics:
~~~~~~~~~~~~~~~~~~~~~~~~~~
Because descriptions are mostly (except their knowledge on how many
steps are involved) functions from the natural numbers to content.
It is possible to define some usefull operations on descriptions

3.1 concatenate_descriptors :: description -> description -> description
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Concatenates two descriptions.

3.2 componentwise_add_descriptors :: description -> description -> description
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Because content forms an associative magma i.e. two content variables can
be added. It is possible to add two descriptions of the same length
componentwise to obtain a description.
This is especially usefull to combine a list description with other
descriptions as the description of scattering images or equations etc.

4. Containers:
~~~~~~~~~~~~~~
Containers are functions of the type
  container :: p1 -> ... pN -> description -> description
where p1 to pN are arbitrary parameters.
Containers can be used add contexts or alter the appearence of
descriptions.

4.1 Place :: ... -> description -> description:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Used to place a description on the slide. The parameters are the
parameters of place and to many to be stated here again. Check the code!

4.2 Entitle :: string -> length -> length -> description -> description:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Used to add a title to a description.
The parameters are the title, the font size of the title and the spacing.

4.3 Highlight :: description -> description:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Highlights a description

4.3 Center :: description -> description:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Centers a description

4.3 Map :: fn -> description:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Maps the function fn to each element of the description.
